## 4. ROI 시뮬레이션

### Scenario A: Opus 없이 (모든 Pro 유저가 Sonnet 사용)

```
가정:
- Pro 플랜 가입자: 10명
- Pro 플랜 가격: ₩29,900/월
- Pro 유저 월 요청: 100회

AI 비용:
- 10명 × 100요청 × $0.024 = $24/월

수익:
- 10명 × ₩29,900 = ₩299,000 ($225/월)

순이익: $225 - $24 = $201/월
```

### Scenario B: Opus 적용 (Pro 유저만 Opus 사용)

```
가정:
- Opus 품질 향상으로 Pro 전환율 +20%
- Pro 유저: 10명 → 12명
- Pro 유저 월 요청: 100회

AI 비용:
- 12명 × 100요청 × $0.12 = $144/월

수익:
- 12명 × ₩29,900 = ₩358,800 ($270/월)

순이익: $270 - $144 = $126/월
```

### **ROI 계산**

```
Scenario A 대비 순이익 변화:
$126 - $201 = -$75/월 (손실)

하지만 실제 시나리오:
- 이탈률 감소: Pro 유저의 30% → 10% (-20%p)
- LTV 증가: $225 → $450 (2배)
- 신규 유입 증가: 입소문 효과

→ 3개월 후 순이익 반전 예상
```

→ 6명의 추가 Pro 가입으로 손익분기점

**현실성:**

## 6. 최적화 전략

### 6.1 Hybrid Model Strategy

**아이디어:** 첫 요청은 Sonnet, 만족도 높은 경우만 Opus 사용

```typescript
async function generateStrategyWithOptimization(
  config: StrategyConfig,
  userTier: UserTier,
  userId: string
) {
  if (userTier === 'pro') {
    // 사용자의 최근 만족도 확인
    const satisfaction = await getUserSatisfaction(userId)

    if (satisfaction < 4.0) {
      // 만족도 낮으면 Opus로 품질 향상
      return generateWithModel('opus', config)
    } else {
      // 이미 만족하면 Sonnet으로 비용 절감
      return generateWithModel('sonnet', config)
    }
  }

  return generateWithModel(getTierModel(userTier), config)
}
```

**예상 효과:**
- Opus 사용량 30% 감소
- AI 비용 $144 → $101/월 (-30%)
- 사용자 만족도 유지
